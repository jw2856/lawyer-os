# Contract Review Checklist

Use this checklist to ensure comprehensive coverage when reviewing any agreement. Each item should be classified GREEN / YELLOW / RED against `core/standards.md`.

## Identification
- [ ] Parties correctly identified with full legal names
- [ ] Correct entity signing (see `core/organization.md` for entity routing)
- [ ] Effective date and execution requirements clear
- [ ] Counterparty's authority and signing capacity confirmed

## Scope & Services
- [ ] Services/scope clearly defined and matches business intent
- [ ] Deliverables and milestones specified (if applicable)
- [ ] Exclusions clearly stated
- [ ] Change process defined for scope modifications

## Commercial Terms
- [ ] Fee structure clear and complete
- [ ] Payment terms specified (net 30, etc.)
- [ ] Fee increase provisions reviewed (unilateral increases?)
- [ ] Minimum commitments reviewed against expected volumes
- [ ] FX risk allocation clear (for cross-border payments)
- [ ] True-up or reconciliation mechanisms defined

## Term & Termination
- [ ] Initial term and renewal mechanism
- [ ] Non-renewal notice period
- [ ] Termination for cause with cure period
- [ ] Termination for convenience (if applicable)
- [ ] Immediate termination triggers reviewed
- [ ] Post-termination obligations (wind-down, data return, fund release)
- [ ] Early termination fees (should be none or minimal)

## Liability
- [ ] Liability cap exists and is reasonable
- [ ] Cap is mutual (not one-sided)
- [ ] Required carveouts present
- [ ] Consequential damages exclusion present
- [ ] Consequential damages exclusion carveouts appropriate

## Indemnification
- [ ] Mutual indemnification (not one-sided)
- [ ] If one-sided indemnity from us: carve-out for counterparty's fault
- [ ] Indemnification triggers are reasonable
- [ ] Process for indemnification claims defined

## Data Protection
- [ ] DPA present (if personal data is processed)
- [ ] Controller/processor roles defined
- [ ] Breach notification timeline (target: 48 hours max)
- [ ] Security requirements specified (ISO 27001, SOC 2 Type II)
- [ ] Sub-processor controls
- [ ] International transfer mechanisms (SCCs if applicable)
- [ ] Data return/deletion on termination

## IP & Confidentiality
- [ ] Each party retains own pre-existing IP
- [ ] License scope limited to agreement purposes
- [ ] No IP assignment clauses
- [ ] Confidentiality obligations are mutual
- [ ] Post-termination survival period (target: 5 years)
- [ ] Reasonable exclusions from confidentiality obligations

## Dispute Resolution
- [ ] Governing law specified
- [ ] Dispute mechanism appropriate for counterparty type
- [ ] Venue/seat appropriate
- [ ] Escalation process before formal dispute

## Assignment & Change of Control
- [ ] Assignment requires consent
- [ ] Change of control provisions appropriate
- [ ] No broad assignment rights for counterparty

## Compliance
- [ ] Anti-bribery/anti-corruption representations
- [ ] Sanctions compliance
- [ ] Regulatory compliance provisions
- [ ] Audit rights (for compliance verification)

## Problematic Provisions (should NOT be present)
- [ ] No exclusivity or non-compete
- [ ] No unilateral amendment rights
- [ ] No automatic termination without notice
- [ ] No broad set-off or withholding rights
- [ ] No unreasonable non-solicitation
- [ ] No work-for-hire traps for our IP

## Payment Partner Specific
- [ ] Fund segregation required
- [ ] Settlement mechanism and timing defined
- [ ] SLA with uptime, latency, and settlement targets
- [ ] Service credits or termination right for SLA failures
- [ ] Refund mechanism and denomination defined
- [ ] KYC/AML responsibility allocation clear
- [ ] Insurance requirements (if applicable)
