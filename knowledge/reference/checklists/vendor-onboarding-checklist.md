# Vendor Onboarding Checklist

## Intake
- [ ] Service/product description documented
- [ ] Internal business owner identified
- [ ] Data access scope determined (personal data, financial, proprietary, none)
- [ ] System access requirements identified
- [ ] Criticality assessment (single point of failure?)

## Risk Tier Classification
- [ ] Tier assigned: HIGH / MEDIUM / LOW
- [ ] Classification rationale documented

## Due Diligence (scale to tier)
- [ ] Security questionnaire or certification reviewed (SOC 2 Type II / ISO 27001)
- [ ] Privacy assessment completed (if personal data involved)
- [ ] DPA reviewed or drafted (if personal data involved)
- [ ] Financial stability assessed (for high-risk vendors)
- [ ] References checked (for high-risk vendors)
- [ ] Insurance verified (if required)

## Agreement Review
- [ ] Contract reviewed per contract-review playbook
- [ ] Key terms acceptable per standards.md
- [ ] Data handling provisions adequate
- [ ] Security requirements specified
- [ ] Termination and data portability provisions adequate
- [ ] SLAs defined (for critical services)

## Approval & Setup
- [ ] Required approvals obtained (based on value/risk tier)
- [ ] Agreement executed
- [ ] Vendor added to tracking system
- [ ] Periodic review date set (annual for high risk, biennial for medium)
